## Rủi ro trong CV cần chuẩn bị

### 1. CV ghi rất nhiều công nghệ

Interviewer có thể test xem bạn thật sự làm sâu hay chỉ liệt kê. Cách xử lý:

- Phân biệt rõ: “I used X hands-on in project Y” vs “I have working knowledge of Z”.
- Không cố trả lời quá mức nếu chỉ biết cơ bản.

### 2. Java SE position nhưng CV senior/cloud-heavy

Có thể bị hỏi Java Core rất kỹ. Cần chuẩn bị HashMap, concurrency, JVM, OOP, exception, generics.

### 3. Con số impact cần giải thích được

Ví dụ “reduced manual processing by 70%” cần biết:

- Manual process trước đây là gì?
- Automation thay đổi flow nào?
- Đo lường 70% như thế nào?

## Câu hỏi nên hỏi ngược interviewer

- Team hiện đang dùng kiến trúc monolith, microservices hay hybrid?
- Backend service chính xử lý workload như thế nào: synchronous API, batch hay event-driven?
- Team đánh giá success của backend engineer/senior engineer bằng tiêu chí nào?
- Có các challenge hiện tại về performance, reliability hoặc maintainability không?
- Quy trình code review, testing và deployment hiện tại như thế nào?
